# Документация: Система гипотез (Hypotheses)

В процессе оптимизации проекта (удаления тяжелых LLM вызовов из `_run_sleep_cycle_sync` - Этап 2) механизм **генерации** гипотез во время сна (REM-фаза) был отключен для экономии ресурсов (CPU).

Однако сам **функционал хранения и проверки гипотез сохранен**, чтобы обеспечить возможность их создания в будущем (например, при бодрствовании или через ручные команды) и поддержания существующей логики доверия.

## Основные компоненты системы гипотез, которые остались:

1.  **Создание временных гипотез (Time Hypothesis):**
    *   Функция: `update_time_hypothesis(user_id: int, status: str)` в `alex_brain.py`.
    *   Суть: Создает и обновляет специфическую гипотезу доверия ("Руслан держит слово и возвращается вовремя"). Эта гипотеза напрямую связана с задержкой возвращения пользователя (функция `process_user_return`).

2.  **Затухание гипотез во сне (Hypotheses Decay Stage):**
    *   Место: Внутри `_run_sleep_cycle_sync` в `alex_brain.py`.
    *   Суть: Во время каждого цикла сна (sync/reset), уверенность (`confidence`) во всех `active` гипотезах снижается на 12% (`new_conf = conf * 0.88`).
    *   Если уверенность падает ниже `0.15`, гипотеза автоматически переходит в статус `refuted` (опровергнута).

3.  **Активная проверка гипотез при бодрствовании:**
    *   Функция: `verify_active_hypotheses(user_id: int, user_text: str, retrieved_memories: list[str])` в `alex_brain.py`.
    *   Суть: Работает в фоновом режиме `post_response_processing` после каждого сообщения от пользователя.
    *   Использует LLM (llama-3.1-8b) для оценки: не подтверждает или не опровергает ли *текущее сообщение пользователя* и *всплывшие воспоминания* какие-либо из активных гипотез бота.
    *   Изменяет статус на `verified` или `refuted` и дает сильный нейрохимический отклик (дофамин, серотонин, норадреналин).

## Зачем это нужно?

Система гипотез формирует механизм "подозрений" и "доверия" бота. Даже без автоматической генерации новых гипотез во сне, механизм необходим для:
*   Обработки временных задержек (пунктуальности) пользователя.
*   Сохранения возможности в будущем добавить генерацию гипотез на лету (например, в `generate_raw_thought_python` если бот замечает несостыковки).

## Как добавить новую гипотезу в код:
Используйте вызов к базе данных:
```python
import database as db
db.add_alex_hypothesis(user_id, "Текст новой гипотезы", confidence=0.5)
```
