# Отчет о проделанной работе: Активная Память и Когнитивная Стабильность Алекса

Этот документ содержит сводное описание последних архитектурных изменений, багфиксов и новых возможностей, внедренных в проект «Эребус» для стабилизации сознания Алекса и снижения нагрузки на контекстное окно API.

---

## 🛠️ 1. Внедрение Активной Памяти (Working Memory / RAM)

Для разгрузки контекстного окна API и исключения потери точных деталей (номера телефонов, имена, пути к файлам) в диалоге был разработан новый гибридный слой памяти — **Рабочая память (Active Memory / RAM)**.

### 💾 А. Структура БД
В SQLite базу данных добавлена таблица `alex_active_memory` с ограничением уникальности по ключу сессии:
```sql
CREATE TABLE IF NOT EXISTS alex_active_memory (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    key TEXT NOT NULL,
    val TEXT NOT NULL,
    confidence REAL DEFAULT 1.0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, key)
);
```

### 🤖 Б. Локальный Regex-препроцессор (CPU-уровень)
В файле `bot.py` перед вызовом основного API-запроса внедрен локальный экстрактор регулярных выражений, который мгновенно извлекает структурированную информацию из реплик пользователя:
* **Телефонные номера**: Паттерн `\+?[78][\s\-]?\(?\d{3}\)?[\s\-]?\d{3}[\s\-]?\d{2}[\s\-]?\d{2}` -> ключ `phone`.
* **Файлы**: Имена файлов с расширениями `.txt`, `.md`, `.py` -> ключ `file_ref`.

### 🧠 В. Двусторонние команды управления (Bilateral Set Commands)
В ядре `alex_brain.py` внедрен парсер скрытых тегов. Алекс может сам в процессе своих рассуждений отправлять команды на запись или удаление фактов из своей RAM:
* `[SET_ACTIVE: key="value"]` (или без кавычек `key=value`) — сохранить факт.
* `[DELETE_ACTIVE: key]` — удалить факт.
* **Скрытие команд**: Эти теги автоматически перехватываются, обновляют БД и полностью вырезаются из текста перед отправкой сообщения в Telegram и сохранением в STM-историю.

### 💤 Г. Консолидация при сне
Во время NREM-фазы сна накопленные переменные из RAM считываются и трансформируются в новые постоянные узлы LTM с генерацией векторных эмбеддингов:
* Ключ `phone` -> конструируется факт: `Я помню точный номер телефона пользователя: [val]` (тип: `biographical`).
* Ключ `file_ref` -> конструируется факт: `Мы ссылались на файл или файлы: [val]` (тип: `semantic`).
* Прочие ключи переводятся как семантические факты о диалоге.
При пробуждении RAM-таблица очищается.

## 🕒 2. Багфикс циклов авто-сна (Constant Sleep Loop)

### 🔍 Проблема
В `reflection_daemon` в `bot.py` текущее время проверялось локально через `datetime.now()` (UTC+3), в то время как SQLite сохраняет временные метки активности в UTC. Из-за этого разница `idle_mins` мгновенно составляла более 180 минут сразу после реплики пользователя. Бот непрерывно запускал асинхронный сон раз в 10 секунд, забивая API-лимиты Groq и вызывая ошибку rate limit (429), из-за чего падали и основные диалоговые функции.

### 🛠 Решение
1. Все сравнения и вычисления дельт времени активности переведены строго на UTC (`datetime.utcnow()`).
2. Внедрен глобальный набор блокировок `active_sleep_users = set()`, исключающий одновременный запуск нескольких циклов сна для одного пользователя.
3. Время в выводе команды `/status` теперь приводится к локальному часовому поясу пользователя с помощью функции `format_utc_to_local`.

## ✉️ 3. Повышение надежности пересылки (`[SEND_TO...]`)

* В системные инструкции промпта `prompts_experiment_chat.py` внесена Лолита (`[SEND_TO_LOLITA]`), чтобы Алекс знал о её существовании и правильном формате команды.
* Парсер перехвата пересылки сообщений переведен с жесткого `re.match` (требовавшего `[` на первом символе строки) на гибкий поиск `re.search` с мягкой нормализацией. Теперь бот корректно перешлет сообщение, даже если модель сделала синтаксическую опечатку или написала префикс `L SEND_TO_LOLITA: "Текст"`.

## 📥 4. Экспорт разума (Full Log Export)

В клавиатуру панели статуса (`/status`) добавлена кнопка **📥 `Экспорт разума (Full Log)`**.
Она делает полный срез текущей когнитивной матрицы Алекса:
* Текущий статус нейробиологии и синаптической усталости.
* Скрытые гипотезы и степень уверенности в них.
* Полный список LTM-узлов и связей графа долговременной памяти.
* Накопленные STM-диалоги всех участников в оперативной памяти.
* История внутренних мыслей и слабый поток.
* Лог химических дельт и триггеров.
Файл отправляется в чат в формате `.txt` документа.
