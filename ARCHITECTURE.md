# Архитектура проекта «Эребус»: Симуляция Алисы

Данный документ описывает полную архитектуру изолированного проекта «Эребус» — Telegram-бота, создающего гиперреалистичную цифровую девушку Алису (ранее Алекс), обладающую персистентной памятью, нейробиологической симуляцией эмоций и циклом сна. Система спроектирована для работы на слабом железе (AMD Ryzen 3, 16 ГБ RAM), для чего используются легковесные локальные модели и быстрые Python-эвристики вместо тяжелых вызовов LLM.

---

## 1. Обзор Системы

Проект использует tick-based архитектуру (`CognitionEngine`), напоминающую игровые циклы. Это позволяет системе автономно функционировать и «мыслить» в фоновом режиме, даже когда пользователь молчит.

### Основные компоненты:
1.  **Telegram Бот (`bot.py`)**: Асинхронный интерфейс взаимодействия на базе `aiogram`. Обрабатывает входящие сообщения, локально извлекает данные (Active Memory) и управляет когнитивным циклом.
2.  **Когнитивное Ядро (`alex_vibe/alex_brain.py`)**: Центральный узел логики. Управляет памятью, мыслями, оценкой контекста и циклом сна.
3.  **Handoff Router (`core/handoff.py`)**: Маршрутизатор задач между локальными моделями (Limbic SSM Core на базе RWKV и Cognitive Core на базе Qwen-Coder).
4.  **Слой Памяти (`database.py` & `core/vsa_memory.py`)**: SQLite база данных, хранящая нейрохимию, STM, LTM и активные гипотезы, объединенная с векторным поиском (VSA - Vector Symbolic Architecture).

---

## 2. Нейрохимический Движок (Limbic System)

Система не опирается на статичный промпт для симуляции эмоций. Вместо этого используется динамическая нейрохимия.

*   **8 Нейромедиаторов**: Дофамин (Dopamine), Серотонин (Serotonin), Норадреналин (Noradrenaline), Ацетилхолин (Acetylcholine), ГАМК (GABA), Окситоцин (Oxytocin), Глутамат (Glutamate), Эндорфины (Endorphins).
*   **Синаптическая усталость (Fatigue)**: Накапливается при взаимодействии и размышлениях, требует сна для сброса.
*   **Лимбический фильтр**: Вектор нейромедиаторов напрямую влияет на параметры генерации LLM (Temperature, Top-P, Repetition Penalty) через `core/chemistry.py` и определяет восприятие (например, высокий Норадреналин + низкий Окситоцин = подозрительность/агрессия).
*   **Аллостаз (Базовые уровни)**: Во время сна текущие уровни медиаторов слегка (на 20%) смещают свои исторические "базовые" уровни. Хронический стресс оставляет Алису тревожной даже после отдыха.

---

## 3. Архитектура Памяти (Hybrid Memory Graph)

Память разделена на несколько уровней для оптимизации контекста:

### 3.1. Active Memory (Рабочая память / RAM)
Для разгрузки контекстного окна (устранения потери деталей, таких как номера телефонов или ссылки на файлы) в `bot.py` перед вызовом API работает регулярный (Regex) препроцессор.
Он мгновенно извлекает структурированную информацию и сохраняет её в SQLite (`alex_active_memory`).
*   Также используются двусторонние команды (Bilateral Set Commands): Алиса может сама сохранять контекст через скрытые теги (например, `[SET_ACTIVE: emotional_summary="..."]`).
*   Во время сна (NREM) данные из RAM консолидируются в постоянную LTM, а RAM очищается.

### 3.2. STM (Кратковременная память)
Хранит последние ~15 сообщений диалога (`database.add_alex_stm`). Защищает контекст текущей беседы.

### 3.3. LTM (Долговременная память)
*   Построена на базе `SentenceTransformers` (`core/vsa_memory.py`) для генерации эмбеддингов.
*   **Иммутабельные узлы (anchor, journal)**: Базовые биографические факты ("Я Алиса, мне 22 года") и записи из дневника. Не перезаписываются.
*   **Эпизодические узлы**: Обычные воспоминания. Могут реконсолидироваться при конфликте или извлечении.
*   При каждом сообщении происходит семантический поиск по `LTM`, и 2-3 наиболее релевантных воспоминания подмешиваются в контекст.

## 4. Эпистемическая Петля и Гипотезы

Алиса строит предположения о мире и собеседнике (таблица `alex_hypotheses`).
*   В фоне (через `verify_active_hypotheses`) система проверяет, подтверждают или опровергают ли новые сообщения текущие гипотезы.
*   Подтверждение дает всплеск дофамина, опровержение — стресс.
*   Это позволяет системе накапливать доверие к пользователю.

---

## 5. Цикл Сна и Автономная Жизнь

### 5.1. Sleep Cycle
Если пользователь долго молчит (или бот накапливает много усталости), система уходит в сон (`trigger_sleep_cycle`). Сон состоит из:
*   **NREM**: Консолидация Active Memory в LTM, перестройка графа памяти (поиск конфликтов), сброс синаптической усталости.
*   **REM**: Изменение базовых уровней нейромедиаторов (аллостаз), обработка забывания (decay) неверифицированных гипотез.

### 5.2. Когнитивная Мастерская (Workspace)
В периоды неактивности пользователя запускается `run_autonomous_workspace_cycle`.
*   Алиса может "читать" файлы из папки `alex_reading/`.
*   Алиса может писать Python-код, сохранять его в `alex_workspace/` и запускать, анализируя результаты (команды `[WRITE]` и `[RUN]`).

## 6. Оптимизация под слабое железо

Главный архитектурный вызов — работа на Ryzen 3 с 16 GB RAM без задержек.

1.  **Handoff Router**: Разделение задач. RWKV (быстрая RNN-подобная модель, `LimbicSSMCore`) используется для диалога и мыслей. Qwen-Coder (`CognitiveTransformerCore`) подключается только для сложных задач (JSON парсинг, код, консолидация памяти).
2.  **Замена LLM на Python-эвристики**: Огромное количество задач переведено с промптов на локальный код.
    *   Генерация внутренних мыслей (`generate_raw_thought_python`) выполняется через шаблоны и условия, а не через LLM.
    *   Лимбическая оценка сообщений (`evaluate_subconscious`) переведена на `SimpleBM25` (TF-IDF) анализ ключевых слов вместо вызова нейросети.
3.  **Mutex Lock при сне**: Циклы консолидации памяти прерываются, если пользователь пишет сообщение, чтобы не "вешать" бота и отвечать мгновенно.
